## Обновления проекта (на 2026-03-08)

Документ фиксирует, что было реализовано сверх или в уточнение к исходному ТЗ `docs/260307-balkanski-kod-tz-mvp.md`.

### 1) Важные изменения в бизнес-логике

- Добавлено поле `solvedAt` в `TrackProgress` для виджета "последние угаданные".
- Уточнено правило правильного ответа:
  - порядок слов не важен;
  - победа засчитывается по названию трека до дефиса (без обязательного указания исполнителя);
  - добавлены fuzzy-механики для частичных и близких словоформ.
- Улучшен partial match:
  - работает мягкое совпадение слов (основа/префикс/небольшие опечатки);
  - сохраняется принцип "частичное совпадение не равно победе", кроме отдельного правила для title-only победы.

### 2) UI/UX улучшения (поверх MVP)

- Главная:
  - share и reset переведены на MUI-диалоги (без `alert/confirm`);
  - статус карточек компактно через Chip при наличии прогресса;
  - сложность отображается звездами;
  - reset-кнопка перенесена вниз (в один ряд с "О проекте").
- Страница трека:
  - кастомный плеер без нативных browser controls;
  - после завершения трека (solved/revealed) доступна прокрутка (seek slider);
  - история попыток в стиле "новые сверху";
  - в историях не дублируется подряд одинаковая попытка;
  - "Карточка результата" показывается только после завершения;
  - кнопка подсказки скрывается, когда все подсказки исчерпаны;
  - добавлена навигация "предыдущий/следующий трек" для in_progress/finished;
  - визуальные акценты состояний:
    - solved: розовый glow,
    - revealed: оранжевый glow,
    - playing (до разгадки): серебристый glow;
  - анимация glow сделана по тени, без "качания" всей карточки.

### 3) Шаринг и ссылки

- Шаринг на главной и в карточке трека переведен на единый формат с preview в диалоге.
- Для Telegram/Web Share добавлен fallback: URL дублируется в `text`, чтобы ссылка не терялась.
- Share URL и локальные mp3 URL учитывают `BASE_URL`/GitHub Pages префикс.

### 4) GitHub Pages + окружение

- Настроен деплой:
  - `gh-pages` dependency,
  - `predeploy` и `deploy` scripts,
  - `vite base = /balkanskikod-quiz/`,
  - `HashRouter` для стабильных deep-links на Pages.
- Добавлены `.env` и `.env.example`:
  - `VITE_APP_NAME`,
  - `VITE_APP_BASE_URL`.
- Тайтл в `index.html` переведен на `%VITE_APP_NAME%`.

### 5) Контент и брендинг

- Заменен favicon на `ico.png`.
- Обновлена страница "О проекте":
  - расширенный текст о концепции проекта;
  - добавлены ссылки на GitHub и Telegram.
- Подготовлен и улучшен промпт-док для генерации новых треков: `docs/hints.md`.

### 6) Техническое качество

- Исправлена изоляция тестов в `storage.test.ts`: улучшен `beforeEach` для корректного сброса mock localStorage между тестами.
- Исключены тестовые файлы из TypeScript компиляции в `tsconfig.app.json` для предотвращения ошибок типизации в тестах.
- Все изменения прогонялись через `npm run lint` и `npm run build`.
- На текущий момент остается предупреждение Vite о размере бандла `> 500kb` (не блокер, но зона для оптимизации).

### 7) Что еще можно улучшить дальше

- Вынести общие share-диалоги в переиспользуемый компонент.
- Добавить lazy-loading/route splitting для снижения bundle size.
- Добавить smoke e2e (минимум: open app -> open track -> submit -> solve/reveal).
