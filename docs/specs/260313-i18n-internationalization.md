# Спецификация: Интернационализация (i18n) проекта Balkanski kod

**Дата:** 2026-03-13  
**Статус:** Готово к реализации  
**Приоритет:** Средний

## 1. Цель задачи

Добавить в проект **Balkanski kod** поддержку мультиязычности для интерфейса пользователя с возможностью переключения между **русским**, **английским** и **сербским** языками.

## 2. Текущее состояние

- Все тексты интерфейса захардкожены на русском языке
- Проект использует React + TypeScript + MUI
- Уже есть многоязычные данные в треках (`names.serbian`, `names.russian`, `names.original`)
- Архитектура построена на FSD + React Contexts
- Используется localStorage для сохранения прогресса

## 3. Требования к реализации

### 3.1. Функциональные требования

1. **Поддерживаемые языки:**
   - Русский (ru) — язык по умолчанию
   - Английский (en)
   - Сербский (sr)

2. **Переключение языка:**
   - Кнопка переключения в шапке приложения
   - Сохранение выбора в localStorage
   - Мгновенное переключение без перезагрузки страницы

3. **Область перевода:**
   - Все UI тексты интерфейса
   - Сообщения об ошибках и обратной связи
   - Meta-теги для SEO
   - Тексты в диалогах (share, give up, reset)
   - Подсказки в игре

### 3.2. Не подлежит переводу:

- Названия треков (уже многоязычные в `track.names`)
- Имена пользователей/прогресс
- URL пути
- Технические термины в логах

### 3.3. Технические требования

1. **Библиотека:** react-i18next
2. **TypeScript:** Полная типизация ключей переводов
3. **Производительность:** Ленивая загрузка переводов
4. **FSD:** Интеграция в существующую архитектуру
5. **Тестирование:** Unit тесты для языкового контекста

## 4. Архитектура решения

### 4.1. Структура файлов (FSD совместимая)

```
src/shared/i18n/
├── locales/
│   ├── ru/
│   │   ├── common.json      # Кнопки, общие элементы
│   │   ├── pages.json       # Тексты страниц
│   │   ├── tracks.json      # Все что связано с треками
│   │   └── meta.json        # Meta-теги
│   ├── en/
│   │   ├── common.json      # Заглушка (копия ru)
│   │   ├── pages.json       # Заглушка (копия ru)
│   │   ├── tracks.json      # Заглушка (копия ru)
│   │   └── meta.json        # Заглушка (копия ru)
│   ├── sr/
│   │   ├── common.json      # Заглушка (копия ru)
│   │   ├── pages.json       # Заглушка (копия ru)
│   │   ├── tracks.json      # Заглушка (копия ru)
│   │   └── meta.json        # Заглушка (копия ru)
│   └── sr-cyrl/             # Сербский кириллицей (СРП)
│       ├── common.json      # Заглушка (копия ru)
│       ├── pages.json       # Заглушка (копия ru)
│       ├── tracks.json      # Заглушка (копия ru)
│       └── meta.json        # Заглушка (копия ru)
├── index.ts                 # Конфигурация i18next
├── types.ts                 # TypeScript типы
└── ui/                      # UI компоненты для i18n
    └── LanguageSelector.tsx # Переиспользуемый переключатель
```

**Примечание по FSD:**
- `shared/i18n/` — правильное расположение для кросс-функциональной логики
- `ui/LanguageSelector.tsx` — переиспользуемый UI компонент
- Экспорт через `index.ts` для соблюдения FSD границ

### 4.2. Контекст языка

```typescript
// src/shared/contexts/LanguageContext.tsx
interface LanguageContextType {
  currentLanguage: 'ru' | 'en' | 'sr' | 'sr-cyrl';
  changeLanguage: (lng: string) => void;
  availableLanguages: Array<{
    code: 'ru' | 'en' | 'sr' | 'sr-cyrl';
    name: string;
    nativeName: string;
  }>;
}
```

### 4.3. Интеграция с существующей архитектурой

1. **App.tsx:** Оборачиваем в `I18nextProvider`
2. **HeaderBlock:** Добавляем переключатель языка после кнопки Share
3. **LanguageSelector:** Переиспользуемый компонент для всех страниц
4. **Все компоненты:** Заменяем hardcoded тексты на `t()`
5. **MetaTags:** Динамические meta-теги в зависимости от языка

### 4.4. Расположение переключателя языка

Переключатель языка размещается в `HeaderBlock.tsx` **после кнопки Share**:

```tsx
// src/pages/home/ui/elements/HeaderBlock.tsx
<Box sx={{ display: 'flex', alignItems: 'center', gap: 1 }}>
  <IconButton onClick={onShareClick}>
    <ShareRoundedIcon />
  </IconButton>
  
  {/* Переключатель языка */}
  <LanguageSelector />
</Box>
```

**Переиспользование на страницах:**
- **HomePage:** HeaderBlock с переключателем
- **TrackPage:** TrackPageHeader с переключателем  
- **AboutPage:** Отдельный заголовок с переключателем

## 8. Реализованные решения

### Технические решения:
1. **Хранение языка:** Новый ключ `balkanski-kod-language` в localStorage
2. **URL язык:** Только UI переключатель, язык по умолчанию — русский
3. **Детектор языка:** Всегда начинаем с русского языка
4. **Fallback стратегия:** Показывать русский текст при отсутствии перевода

### Контентные решения:
5. **Сербский перевод:** Все языковые файлы создаются как заглушки (копии ru)
6. **Адаптация текстов:** Только заглушка в этой задаче, без культурной адаптации
7. **Длина текстов:** Адаптация верстки при тестировании

### UI/UX решения:
8. **Переключатель языка:** Дропдаун с текстом (ENG, РУС, SRB, СРП)
9. **Положение переключателя:** В шапке после кнопки Share
10. **Индикатор языка:** Показывать текущий язык постоянно

### Приоритетные решения:
11. **Язык по умолчанию:** Русский
12. **Порядок реализации:** Сразу все 4 варианта с заглушками
13. **Тестирование:** Самостоятельная проверка переводов
