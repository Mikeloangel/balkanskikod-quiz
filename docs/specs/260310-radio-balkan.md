# ТЗ: Balkanski kod Radio

## Общее описание

Добавить в приложение "Balkanski kod" виджет радио "Balkanski kod Radio" - непрерывное воспроизведение треков из игры в фоновом режиме. Радио доступно на всех страницах приложения и продолжает играть при навигации между страницами.

## Функциональные требования

### 1. Виджет радио
- Расположение: фиксированный виджет внизу экрана на всю ширину
- Адаптивный дизайн: компактный на мобильных устройствах
- Содержит:
  - Кнопка Play/Pause
  - Регулятор громкости
  - Название текущего трека (оригинал + сербский вариант)
  - Кнопка "Открыть трек в Suno"
  - Прогресс-бар воспроизведения
  - Текущее время / общая длительность

### 2. Управление воспроизведением
- **При первом посещении**: радио не играет, кнопка в состоянии "Play"
- **Запуск**: при нажатии Play начинается воспроизведение с первого трека или сохраненной позиции
- **Пауза**: при нажатии Pause воспроизведение приостанавливается
- **Автопереход**: после окончания трека автоматически воспроизводится следующий
- **Ограничение по времени**: радио автоматически ставится на паузу после 30 минут воспроизведения (в конце текущего трека)
- **Обработка ошибок**: при ошибке загрузки трека - автоматически пропустить следующий

### 3. Сохранение состояния
- Сохранять ID текущего трека и время начала воспроизведения в localStorage
- При следующем посещении продолжать воспроизведение с сохраненной позиции
- Если сохраненный трек отсутствует в списке - начинать с первого
- Состояние сохраняется отдельно от игрового прогресса

### 4. Взаимодействие с игровым плеером
- **При запуске игрового трека**: радио автоматически ставится на паузу
- **Ручное возобновление**: пользователь может продолжить слушать радио нажав кнопку Play в виджете
- **Автоматическое возобновление запрещено**: радио не начинает играть автоматически после остановки игрового трека

### 5. Навигация между страницами
- Радио продолжает играть при смене страниц (HomePage, TrackPage, AboutPage)
- Состояние воспроизведения и текущий трек сохраняются
- Виджет остается видимым на всех страницах

## Технические требования

### 1. Архитектура
- Следовать FSD (Feature-Sliced Design)
- Компонентный подход, без файлов более 300 строк
- Использовать React Context для управления состоянием радио
- Отдельный AudioProvider для радио (доступен при смене роутов)
- Интеграция с существующей архитектурой приложения

### 2. Источники данных
- Создать новый файл `src/shared/config/radioTracks.ts` на основе `src/shared/config/tracks.ts`
- Использовать те же поля `names.original` и `names.serbian` для отображения
- Использовать поле `links.suno` для кнопки открытия в Suno
- Структура `RadioTrack` аналогична `Track`, но без игровых полей (`hints`, `difficulty`)

### 3. localStorage
- Новый ключ для хранения состояния радио (отдельный от игрового прогресса)
- Сохранять: `currentTrackId`, `playbackStartTime`, `totalPlayedTime`
- Обратная совместимость при изменении структуры
- Константа для ограничения времени воспроизведения через env (`VITE_RADIO_MAX_PLAY_TIME_MINUTES = 30`)

### 4. UI/UX
- Dark mode (соответствует текущему дизайну)
- Адаптивный дизайн: компактный виджет на мобильных устройствах
- Использование Material-UI компонентов
- Плавные анимации переходов
- Интуитивные иконки для кнопок

### 5. Аналитика
- Интеграция с Clarity для отслеживания прослушиваний радио
- Метки: `radio-play`, `radio-pause`, `radio-track-change`, `radio-error`

### 6. Производительность
- Минимальное влияние на производительность основного приложения
- Эффективная работа с localStorage
- Правильная очистка ресурсов при unmount

## Интеграционные требования

### 1. Взаимодействие с AudioProvider
- При запуске игрового трека в TrackPage: радио должно ставить на паузу
- Отдельный RadioProvider для радио (доступен глобально)
- Независимое управление громкостью для радио и игрового плеера

### 2. Router интеграция
- Виджет должен работать с HashRouter
- Сохранять состояние при навигации

### 3. Тестирование
- Unit тесты для контекста радио (аналогично существующим AudioContext тестам)
- Unit тесты для storage функций
- Интеграционные тесты для взаимодействия с игровым плеером
- Тесты localStorage функций
- Тесты ограничения по времени воспроизведения

## MVP требования

**Критически важный функционал для первой версии:**
- Базовый виджет с Play/Pause и отображением трека
- Сохранение состояния в localStorage
- Автоматическая пауза при запуске игрового трека
- Переход между треками
- Ограничение по времени (30 минут)
- Обработка ошибок загрузки

## Дизайн и UX

**Виджет радио (desktop):**
- Фиксированное позиционирование внизу экрана
- Высота ~80px, темная тема с полупрозрачностью
- Слева: Play/Pause кнопка
- Центр: информация о треке (оригинал / сербский)
- Справа: регулятор громкости, кнопка Suno, прогресс-бар

**Виджет радио (mobile):**
- Компактный режим, скрытие второстепенных элементов
- Крупные кнопки для touch-взаимодействия
- Минимальная высота ~60px
